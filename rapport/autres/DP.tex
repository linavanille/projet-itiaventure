\chapter{Design Pattern}

Dans le cadre de la refonte architecturale du projet ITIAventure, nous avons introduit le design pattern \textbf{Factory}. Ce patron de création a été choisi pour séparer clairement la logique d’instanciation des entités (comme les pièces, portes ou créatures) du reste de l’application. En centralisant cette logique dans une classe dédiée, le code devient plus modulaire, maintenable et extensible, facilitant notamment l’ajout de nouveaux univers de jeu.

\section{Objectifs du pattern}

Le pattern \textit{Factory} a pour vocation de proposer une interface standardisée pour la création d’objets, tout en dissimulant les classes concrètes qui les implémentent. Ce découplage permet à l’application de rester indépendante des détails de construction des objets manipulés.

Dans notre cas, ce besoin s’est exprimé lors de la lecture du fichier de description du labyrinthe. Initialement, la classe \texttt{LecteurDescription} invoquait directement les constructeurs des entités, ce qui liait fortement cette classe à une version particulière de l’univers du jeu.

\section{Mise en œuvre}

Nous avons conçu une classe \texttt{ITIAventureFactory} regroupant les méthodes de création des entités suivantes :
\begin{itemize}
  \item \texttt{creationMonde()} $\rightarrow$ \texttt{Monde}
  \item \texttt{creationPiece()} $\rightarrow$ \texttt{Piece}
  \item \texttt{creationPorte()} $\rightarrow$ \texttt{Porte}
  \item \texttt{creationMonstre()} $\rightarrow$ \texttt{Monstre}
  \item \texttt{creationCle()} $\rightarrow$ \texttt{Cle}
  \item \texttt{creationSerrure()} $\rightarrow$ \texttt{Serrure}
\end{itemize}

La classe \texttt{LecteurDescription} a été modifiée afin de déléguer la création de ces entités à la factory, plutôt que de les instancier directement. Ce changement rend possible la sélection dynamique de la factory à utiliser, en se basant sur le contenu du fichier de description (notamment sa première ligne).

\section{Extension à un nouvel univers}

Dans le but de proposer une version alternative de l’univers du jeu, nous avons introduit de nouvelles entités propres à un contexte de science-fiction. Ces classes ont été développées dans le package \texttt{fr.insarouen.iti.prog.aventure.spaceopera}, chacune héritant de leur équivalent dans l’univers d’origine :
\begin{itemize}
  \item \texttt{Galaxie} (héritant de \texttt{Monde})
  \item \texttt{VaisseauSpatial} (héritant de \texttt{Piece})
  \item \texttt{Teleporteur} (héritant de \texttt{Porte})
  \item \texttt{Alien} (héritant de \texttt{Monstre})
  \item \texttt{Badge} (héritant de \texttt{Cle})
  \item \texttt{LecteurBadge} (héritant de \texttt{Serrure})
\end{itemize}\\

\vspace{0.5cm}

Afin d’intégrer ces entités dans notre moteur de jeu tout en conservant la logique de création centralisée, nous avons implémenté une classe \texttt{ITISpaceOperaFactory}, conforme à l’interface définie par la factory abstraite. Cette classe redéfinit les méthodes de création telles que :
\begin{itemize}
  \item \texttt{creationMonde()} $\rightarrow$ \texttt{Galaxie}
  \item \texttt{creationPiece()} $\rightarrow$ \texttt{VaisseauSpatial}
  \item \texttt{creationPorte()} $\rightarrow$ \texttt{Teleporteur}
  \item \texttt{creationMonstre()} $\rightarrow$ \texttt{Alien}
  \item \texttt{creationCle()} $\rightarrow$ \texttt{Badge}
  \item \texttt{creationSerrure()} $\rightarrow$ \texttt{LecteurBadge}
\end{itemize}\\
